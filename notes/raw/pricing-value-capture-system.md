# Pricing Value Capture System

## 問題定義
- 定價策略常被拆成零散決策：成本、競品、折扣、銷售話術彼此脫節。
- 結果是「成交了但沒賺到合理利潤」，或「毛利保住了但轉換率掉太多」。

## 系統目標
建立一個可重複、可校正的價值擷取系統，讓團隊在不依賴單一高手的情況下，持續提升：
1. 平均成交單價（ASP）
2. 毛利率（Gross Margin）
3. 價格實現率（Price Realization）
4. 折扣紀律（Discount Governance）

## 五層架構

### 1) Segmentation（分群）
- 依產業、使用情境、急迫程度、替代方案成本分群。
- 每個分群定義不同的願付價格區間（WTP band）。

### 2) Packaging（包裝）
- 將功能與服務打包成 Good / Better / Best 三層。
- 把「高價值、低邊際成本」項目優先放在高階方案。

### 3) Price Metric（計價單位）
- 選擇與客戶價值成長同向的計價基礎（例如：每席次、每使用量、每結果）。
- 避免與客戶價值脫鉤的固定費模型。

### 4) Guardrails（護欄）
- 設定底價、可授權折扣區間、超額折扣審批流程。
- 讓折扣成為「有條件交換」而非自動讓利。

### 5) Learning Loop（學習迴圈）
- 每月回顧贏單/輸單原因、價格異議、折扣分佈。
- 迭代分群、包裝與報價話術，形成組織資產。

## 90 天落地節奏
- Day 1-30：盤點歷史報價與折扣資料，建立分群與基準價。
- Day 31-60：上線新方案包裝與折扣護欄，訓練銷售話術。
- Day 61-90：建立月度檢討機制，追蹤 ASP、毛利率、價格實現率。

## 成功指標
- ASP 提升 8-15%
- 無理由折扣案下降 30%+
- 同等成交量下毛利額成長
